# ReplaceWithContent — вставка содержимого документа по текстовому тегу

## Обзор

Метод `ReplaceWithContent` заменяет все вхождения текстового тега (placeholder) в документе-шаблоне на содержимое другого `.docx` документа. Обрабатывает тело документа, все колонтитулы (верхние/нижние) и комментарии.

```go
func (d *Document) ReplaceWithContent(old string, cd ContentData) (count int, err error)
```

**Параметры:**
- `old` — текстовый тег для поиска (например `<mark>`, `{{content}}`). Пустая строка — no-op, возвращает 0.
- `cd` — структура `ContentData`, описывающая источник и параметры импорта.

**Возвращает:**
- `count` — количество выполненных замен (во всех контейнерах: тело + колонтитулы + комментарии).
- `err` — ошибка. При ошибке тело документа автоматически откатывается к состоянию до вызова (snapshot/rollback). Ресурсы (стили, нумерация), импортированные до ошибки, остаются в документе, но безвредны — Word игнорирует неиспользуемые определения.

---

## ContentData

```go
type ContentData struct {
    Source  *Document
    Format  ImportFormatMode
    Options ImportFormatOptions
}
```

### Source

Открытый документ-источник. Из его `<w:body>` извлекаются блочные элементы (параграфы `<w:p>`, таблицы `<w:tbl>`) для вставки. Секционные свойства (`<w:sectPr>`), колонтитулы и заголовки исходного документа **не включаются**.

Документ-источник должен оставаться открытым до завершения вызова `ReplaceWithContent`. После вызова его можно закрыть.

### Format (ImportFormatMode)

Стратегия разрешения конфликтов стилей между исходным и целевым документами.

```go
type ImportFormatMode int

const (
    UseDestinationStyles ImportFormatMode = iota  // 0 — по умолчанию
    KeepSourceFormatting                          // 1
    KeepDifferentStyles                           // 2
)
```

#### UseDestinationStyles (по умолчанию)

При конфликте стилей (одинаковый `styleId` в обоих документах) используется определение стиля из **целевого** документа. Стили, существующие только в источнике, копируются вместе с цепочкой зависимостей (`basedOn`, `next`, `link`).

**Когда использовать:** целевой документ (шаблон) — эталон форматирования. Вставляемый контент адаптируется под стили шаблона.

```go
cd := docx.ContentData{Source: markDoc}
// или явно:
cd := docx.ContentData{Source: markDoc, Format: docx.UseDestinationStyles}
```

#### KeepSourceFormatting

Визуальное форматирование исходного документа сохраняется. При конфликте стилей:

- **По умолчанию** (`ForceCopyStyles: false`): свойства стиля источника разворачиваются в прямые атрибуты параграфа/run (`<w:pPr>`, `<w:rPr>`), ссылка на стиль меняется на стиль по умолчанию целевого документа.
- **С `ForceCopyStyles: true`**: стиль источника копируется в целевой документ с уникальным суффиксом (`_0`, `_1`, ...) для избежания коллизии.

**Когда использовать:** вставляемый контент должен выглядеть точно как в оригинале, независимо от стилей шаблона.

```go
cd := docx.ContentData{
    Source: markDoc,
    Format: docx.KeepSourceFormatting,
}
```

#### KeepDifferentStyles

Гибридная стратегия. Для каждого конфликтующего стиля:

- Если определения в источнике и цели **идентичны** → используется стиль цели (как `UseDestinationStyles`).
- Если определения **различаются** → поведение как `KeepSourceFormatting` (разворачивание в прямые атрибуты или копирование с суффиксом при `ForceCopyStyles`).

**Когда использовать:** компромисс между двумя предыдущими режимами. Минимизирует изменения, сохраняя различия.

```go
cd := docx.ContentData{
    Source: markDoc,
    Format: docx.KeepDifferentStyles,
}
```

### Options (ImportFormatOptions)

Тонкая настройка поведения импорта. Все поля по умолчанию `false`.

```go
type ImportFormatOptions struct {
    ForceCopyStyles    bool
    KeepSourceNumbering bool
}
```

#### ForceCopyStyles

Принудительное копирование конфликтующих стилей в целевой документ с уникальным суффиксом (`_0`, `_1`, ...) вместо разворачивания форматирования в прямые атрибуты.

Работает только с `KeepSourceFormatting` и `KeepDifferentStyles`. С `UseDestinationStyles` не имеет эффекта.

```go
cd := docx.ContentData{
    Source:  markDoc,
    Format:  docx.KeepSourceFormatting,
    Options: docx.ImportFormatOptions{ForceCopyStyles: true},
}
```

#### KeepSourceNumbering

Сохранение нумерации списков из исходного документа как отдельных определений. При `false` (по умолчанию) исходные списки могут объединяться с совпадающими списками в целевом документе.

> **Примечание:** текущая реализация всегда создаёт отдельные определения списков (эквивалент `KeepSourceNumbering: true`).

## Что импортируется

| Ресурс | Поведение |
|--------|-----------|
| **Параграфы и таблицы** | Блочные элементы `<w:body>` копируются. Секционные `<w:sectPr>` исключаются. |
| **Стили** | BFS-замыкание по зависимостям (`basedOn`, `next`, `link`). Стратегия зависит от `Format`. |
| **Нумерация (списки)** | Определения `<w:abstractNum>` и `<w:num>` импортируются с новыми ID. |
| **Сноски и концевые сноски** | Импортируются с новыми ID. Тела сносок обрабатываются для стилей, нумерации и связей. |
| **Изображения** | Копируются в целевой пакет с дедупликацией по SHA-256 хешу. |
| **Гиперссылки** | Внешние связи (`r:id`) переносятся в целевой документ. |
| **Диаграммы, VML, embedded-части** | Глубокое копирование с рекурсивным импортом подчинённых связей. |
| **Аннотации** | Удаляются (закладки, комментарии, трекинг изменений) — они привязаны к источнику. |

## Компенсация docDefaults

Когда исходный и целевой документы имеют разные `<w:docDefaults>` (базовый шрифт, размер, межстрочный интервал), скопированные стили неявно наследуют значения из **целевого** документа вместо исходного. Это приводит к визуальным различиям.

Механизм компенсации:

1. **Вычисление дельты** (`computeDocDefaultsDelta`): разница между `docDefaults` источника и цели.
2. **Инъекция дельты** в скопированные стили: недостающие свойства из `docDefaults` источника явно прописываются в определения стилей.
3. **Три стратегии компенсации** для каждого скопированного стиля:
   - `compensateAll` — корневой стиль (без `basedOn`): инжектируется вся дельта.
   - `compensateNone` — родительский стиль тоже скопирован: пропуск (транзитивное покрытие).
   - `compensateUncovered` — родитель в целевом документе: инжектируются только свойства, не покрытые цепочкой стилей.

## Конвейер обработки

```
ReplaceWithContent(old, cd)
│
├─ Phase 1: Импорт ресурсов (один раз на вызов)
│  ├─ importNumbering()      — нумерация списков
│  ├─ importStyles()         — BFS-замыкание стилей + компенсация docDefaults
│  ├─ importFootnotes()      — сноски
│  └─ importEndnotes()       — концевые сноски
│
├─ Phase 2: Подготовка контента
│  ├─ Deep-copy элементов из <w:body> источника
│  ├─ sanitizeForInsertion() — удаление аннотаций и sectPr
│  ├─ materializeImplicitStyles() — явная простановка стилей по умолчанию
│  ├─ expandDirectFormatting()    — разворачивание стилей в прямые атрибуты
│  ├─ remapAll()                  — перезапись ссылок на стили/нумерацию/сноски
│  └─ importRelationships()       — импорт связей (изображения, гиперссылки)
│
├─ Phase 3: Вставка в целевой документ
│  ├─ Тело документа (body)
│  ├─ Колонтитулы (headers/footers) — с дедупликацией
│  └─ Комментарии (comments)
│
└─ Возврат count, err
```

Каждая вставка создаёт независимую глубокую копию подготовленных элементов, включая перенумерацию drawing-идентификаторов (`docPr`, `cNvPr`) для уникальности в рамках story-part.

## Примеры использования

### Базовая вставка (стили шаблона)

```go
templateDoc, _ := docx.Open("template.docx")
markDoc, _ := docx.Open("content.docx")

count, err := templateDoc.ReplaceWithContent("<mark>", docx.ContentData{
    Source: markDoc,
})
fmt.Printf("Заменено: %d\n", count)

templateDoc.Save("result.docx")
```

### Сохранение форматирования источника

```go
count, err := templateDoc.ReplaceWithContent("<mark>", docx.ContentData{
    Source: markDoc,
    Format: docx.KeepSourceFormatting,
})
```

### Принудительное копирование стилей

```go
count, err := templateDoc.ReplaceWithContent("<mark>", docx.ContentData{
    Source: markDoc,
    Format: docx.KeepSourceFormatting,
    Options: docx.ImportFormatOptions{
        ForceCopyStyles: true,
    },
})
```

### Гибридный режим

```go
count, err := templateDoc.ReplaceWithContent("<mark>", docx.ContentData{
    Source: markDoc,
    Format: docx.KeepDifferentStyles,
    Options: docx.ImportFormatOptions{
        ForceCopyStyles: true,
    },
})
```

## Обработка ошибок

- `cd.Source == nil` → ошибка `docx: ContentData.Source is nil`.
- `old == ""` → no-op, возвращает `(0, nil)`.
- Ошибка на любом этапе → тело документа откатывается (snapshot). Стили/нумерация/сноски, импортированные до ошибки, остаются, но не влияют на документ.
- Битые связи в источнике (`r:id` без целевой части) → пропускаются молча.
- Циклические зависимости стилей (`A → B → A`) → защита через `visited` set, бесконечный цикл невозможен.

---

## Ограничения

- Секционные свойства (`<w:sectPr>`) источника не переносятся — вставка не создаёт разрывы секций.
- Аннотации источника (закладки, комментарии, трекинг изменений) удаляются.
- `KeepSourceNumbering` в текущей реализации всегда `true` (отдельные определения списков).
- Одновременный вызов `ReplaceWithContent` на одном документе из разных горутин **не поддерживается** — `ResourceImporter` не потокобезопасен.
